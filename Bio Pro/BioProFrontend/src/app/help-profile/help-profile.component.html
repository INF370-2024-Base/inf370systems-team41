history Audit Trail

This section provides assistance on how to view, filter, and interpret the audit trail for system transactions.

Common Questions

  • What is an audit trail, and why is it important?

    An audit trail is a comprehensive log that records all transactions and actions within the system, capturing essential data such as the user who performed the action, the type of transaction (e.g., Create, Update, Delete), and additional data related to the transaction. It is crucial for monitoring system activity, ensuring accountability, and tracking changes for security purposes.

  • How do I view the complete audit trail in the system?

    To view the audit trail, navigate to the 'Audit Trail' page from the main navigation menu. Once on the page, you will see a table listing all transactions, with columns such as 'Audit Trail ID', 'Date of Transaction', 'System User', 'Transaction Type', and 'Additional Data'. These columns provide detailed information about each system action.

  • Can I filter the audit trail to see transactions from a specific date range?

    Yes, you can filter the audit trail by selecting a 'Start Date' and an 'End Date'. This filter allows you to narrow down the list of transactions to only those that occurred within the specified date range. Use the date pickers to select the desired dates, and the audit trail will update automatically.

  • How can I view audit trail entries for a specific user?

    To view the transactions performed by a specific user, use the 'User' dropdown filter on the audit trail page. This will display a list of unique users, identified by their email addresses, and allow you to select a specific user. Once selected, the table will filter to show only the actions performed by that user.

  • What does the 'Transaction Type' refer to, and how can I use it to filter results?

    The 'Transaction Type' represents the kind of action taken by the user (e.g., 'Create', 'Update', 'Delete'). You can filter the audit trail by selecting a specific transaction type from the 'Transaction Type' dropdown, enabling you to focus on only certain kinds of system activities, such as updates or deletions.

  • What information is available in the 'Additional Data' column?

    The 'Additional Data' column provides further context about the transaction, such as what record was affected, the changes made, or any other details that were part of the action. This data is critical for understanding the full scope of the action performed and can help in auditing the exact nature of the change.

  • How do I reset the filters if I want to see the full audit trail again?

    If you want to clear all filters and return to viewing the complete list of audit trail entries, you can click the 'Clear Filters' button. This will reset the 'Start Date', 'End Date', 'User', and 'Transaction Type' filters, allowing you to see all transactions logged in the system without restrictions.

  • How can I sort the audit trail data by date or other columns?

    You can sort the audit trail by clicking on the headers of any column. For example, clicking on the 'Date of Transaction' column header will sort the table by date in ascending or descending order, allowing you to quickly find the most recent or oldest transactions. This functionality also applies to the other columns, such as 'System User' and 'Transaction Type'.

  • Can I export the audit trail for external review?

    Currently, the system does not provide an export function directly through the UI. If you need to export the audit trail data for external review or backup purposes, contact your system administrator, who may be able to assist with exporting the data directly from the database or back-end system.
